## Bugs gefunden & behoben

### Bug 1: button_validate zu früh aufgerufen
- **Problem:** Nach dem Bestätigen der ersten Pick-Zeile wurde das Picking sofort auf `done` gesetzt — alle weiteren Zeilen verschwanden.
- **Ursache:** `confirm_pick_line()` in `picking_service.py` prüfte `all_done` mit einem `or`-Fallback: `move.get("quantity_done", 0) >= move.get("quantity", 0)`. Da die geladenen Felder nur `["picked"]` enthielten, waren `quantity_done` und `quantity` immer `0` → `0 >= 0 = True` → alle Moves galten als fertig.
- **Fix:** Fallback entfernt. Nur noch `all(move.get("picked") for move in moves)`.
- **Status:** ✅ Gefixt, Tests aktualisiert

### Bug 2: Voice-Modus blieb bei Navigation aktiv
- **Problem:** M-Taste oder Mic-Button aktivierten den Voice-Modus in der Detail-Ansicht. Beim Navigieren (zurück zur Liste, Quality-Alert öffnen) blieb das Mikrofon aktiv, obwohl der Button versteckt war.
- **Ursache:** `document.addEventListener('keydown', ...)` ist global und wird nie entfernt. `updateToolbar()` versteckte zwar den Button, stoppte aber den Voice-Modus nicht.
- **Fix:** `stopVoiceMode()` in `voice.js` exportiert. In `updateToolbar(view)` wird `stopVoiceMode()` aufgerufen wenn `view !== 'detail'`.
- **Status:** ✅ Gefixt

## Neue Features

### Voice-First UI
- **Proaktive Begrüßung:** App spricht beim Öffnen automatisch: "X Aufträge offen. Y davon dringend."
- **Prioritäts-Filter:** Buttons `[Alle]` und `[⚡ Dringend]` in der Picking-Liste. Filtert Pickings nach Odoo-Priority `'2'` oder `'3'`. Count-Badge rechts: "2 von 7".
- **Voice-Intents für Filter:**
  - "dringend" / "hohe Priorität" / "kritisch" → `filter_high` → Filter aktiv
  - "alle" / "zurücksetzen" → `filter_normal` → Filter weg
  - "wie viele" / "status" / "übersicht" → `status` → spricht Anzahl offen + dringend

### Neue Voice-Kommandos
| Befehl | Intent | Aktion |
|--------|--------|--------|
| "foto" / "kamera" | `photo` | Kamera-Scanner öffnen |
| "pause" / "stopp" | `pause` | Voice-Modus deaktivieren |
| "fertig" | `done` | Wenn alle bestätigt → zur Liste; sonst "Noch X ausstehend" |
| "dringend" | `filter_high` | Nur dringende Pickings zeigen |
| "alle" / "zurücksetzen" | `filter_normal` | Filter aufheben |
| "wie viele" / "status" | `status` | Anzahl offener Aufträge ansagen |
| "noch da" / "lagerbestand" | `stock_query` | Lagerbestand für aktuelles Produkt abfragen |

### Audio-Feedback (`feedback.js`)
- Rein Web Audio API — kein externes Asset, kein Server
- `feedbackSuccess()` → heller Doppel-Piep (880Hz → 1100Hz) + kurze Vibration `[50,30,50]`
- `feedbackError()` → tiefer Brummton (180Hz, Square-Wave) + lange Vibration `[200,100,200]`
- `feedbackAlert()` → markante 3-Ton-Folge (für spätere Eilauftrag-Benachrichtigung)
- Ausgelöst bei Scan: Erfolg → `feedbackSuccess`, falsch/Fehler → `feedbackError`

### Bestandsabfrage
- **Neuer Endpoint:** `GET /api/pickings/{id}/stock?product_id=X&location_id=0`
- **Funktion:** Fragt `stock.quant` in Odoo ab, gibt `quantity_available` + `quantity_total` zurück
- **Voice:** "Noch da?" → Backend → "Laut System sind 5 Stück verfügbar." oder "Kein Bestand. Soll ich einen Qualitätsalarm auslösen?"
